Zdarzenia

Po co są zdarzenia?

Kiedy napiszemy naszą stronę internetową z użyciem HTML i CSS jest ona statyczna czyli użytkownik nie może wchodzić z nią w interakcję. Przykładem zdarzenia jest na przykład kliknięcie przycisku czy najechanie kursorem na jakiś element.

Za podstawowy rodzaj zdarzenia przyjmijmy kliknięcie przycisku. Możemy je zaimplementować na dwa sposoby. Pierwszym z nich jest onclick - w przycisku umieszczamy wyrażenie onclick i po nim nazwę funkcji, która wykona się po kliknięciu tego przycisku. 


        function funkcja(){
            //kod, który wykona się po wciśnięciu przycisku
            }

Drugim sposobem jest dodanie event listenera, czyli funkcji nasłuchującej jakiegoś zdarzenia na podanym elemencie.


        const button = document.querySelector('.default-button');
        button.addEventListener("click", ()=>{
            //kod, który wykona się po wciśnięciu przycisku
            });

Zaletą event listenera jest to, że jeśli chcemy zmienić zdarzenie, albo nazwę funkcji, musimy edytować jedynie JS.


Wykorzystując zdobytą wiedzę wykonaj poniższe zadanie:

Na stronie umieść wyśrodkowany paragraf o treści: "Twoje imię: " oraz przycisk z napisem "Kliknij mnie!", po którego kliknięciu:

  1. Zmieni się jego kolor
  2. Pod spodem pojawi się napis: "Kliknąłeś przycisk!"
  3. W paragrafie "Twoje imię" dopisze się Twoje imię
  4. Zmieni się napis na przycisku na "Kliknięty!"
